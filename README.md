# Upkeep

Two apps, one Supabase backend.

- **`apps/mobile`** — Expo / React Native app for cleaners. Shows what's assigned to me today; lets me start and complete tasks.
- **`apps/web`** — Next.js 16 admin web app. 3D house, drag-drop assignment, photo completion proof.
- **`packages/shared`** — TypeScript types shared between both apps (mirrors the Supabase schema).
- **`supabase/migrations`** — SQL applied to Supabase. Source of truth for the schema.

## First-time setup

```bash
# Install everything (npm workspaces hoist deps to the root).
npm install
```

Then for each app, copy the env example and fill in your Supabase project's URL + anon key:

```bash
cp apps/web/.env.local.example apps/web/.env.local
cp apps/mobile/.env.local.example apps/mobile/.env.local
```

Get those values from **Supabase Dashboard → Project Settings → API**.

## Apply the schema

Open **Supabase Dashboard → SQL Editor**, paste the contents of
`supabase/migrations/0001_initial_schema.sql`, and run it. It creates the
tables, RLS policies, profile-creation trigger, seed rooms / task
templates, and the `task-photos` storage bucket.

After your first user signs up, promote them to admin from the SQL Editor:

```sql
update public.profiles set role = 'admin'
where id = (select id from auth.users where email = 'you@example.com');
```

## Running the apps

From the repo root:

```bash
npm run web          # next dev — admin web app at http://localhost:3000
npm run mobile       # expo start — scan QR with Expo Go or open in simulator
npm run mobile:ios   # expo run:ios — build a dev client and install on simulator
```

## Typecheck

```bash
npm run typecheck    # runs tsc --noEmit in every workspace that has the script
```

## Schema model

```
profiles         (id = auth.users.id, role: admin|cleaner)
rooms            (Kitchen, the Symposium, bathrooms, …)
task_templates   (reusable defs: title, category, optional room_id, est. minutes)
task_instances   (one row per template+day: assignee, status, photo_url, scheduled_for)
```

## Conventions

- The mobile app's existing Inventory and Laundry features are dormant (no
  tab, code retained). Re-enable once the admin assignment loop ships.
- Native iOS project (`apps/mobile/ios/`) is gitignored and regenerated by
  `expo prebuild` when needed.
- Don't commit `.env.local`. The `.env.local.example` files are the
  committed templates.
